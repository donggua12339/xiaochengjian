# 安全披露政策

## 报告漏洞

如果你发现安全漏洞,请**不要**公开 Issue,通过以下渠道私密披露:

1. **GitHub Security Advisory**(推荐):https://github.com/xiaochengjian/xiaochengjian/security/advisories/new
2. **邮箱**:security@xcj.example.com

## 响应 SLA(ADR 0057)

| 阶段 | 时间 |
|---|---|
| 确认收到 | 48 小时内 |
| 初步评估 | 7 天内 |
| 修复发布 | 90 天内(严重漏洞优先) |
| 公开公告 | 修复发布后 |

## 报告格式

请包含:

- 漏洞描述 + 影响范围
- 复现步骤(最小化)
- 影响版本
- 建议修复方案(可选)

## 负责任披露

- 鼓励负责任披露,不鼓励公开漏洞(修复前)
- 修复后我们会致谢报告者(如同意)
- 严禁利用漏洞获取未授权数据
