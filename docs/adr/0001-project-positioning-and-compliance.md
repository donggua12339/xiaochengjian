# ADR 0001 · 项目定位与合规红线

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:战略

## 背景

小城笺是类似"小纸片/大纸片/Arm Pro/木华"的卡密验证系统,行业存在被滥用于外挂、私服、破解工具的风险。需要在项目立项时明确合规边界,避免项目变性质。

## 决策

### 项目定位
- 名称:小城笺(Xiaochengjian)
- 形态:开源 + SaaS 双模式 Android 卡密验证系统
- 目标用户:P0 个人 Android 应用开发者(MVP 仅 Android)
- 价值主张:保障创作者权益,杜绝付费应用盗版
- 商业模式:开发者订阅(月度 + 终身 + VIP 解锁功能)

### 合规红线
**允许:**
- 开发者对自己开发的付费应用做版权保护、防盗版验证
- 试用/付费转化、设备绑定、卡密销售

**禁止:**
- 用于外挂、私服、破解工具、规避反作弊、赌博、诈骗
- 主动提供"绕过其他验证系统"的功能
- 注入他人 APK 用于商业转售(注入工具仅限开发者自有 APK)

### 合规执行方式
- 协议约束:用户协议明确红线,用户使用即视为同意
- 不强制实名认证(参考 Arm Pro 做法,降低接入门槛)
- 不做过载检查(减轻工作量,依靠法律约束 + 追溯机制)
- 发现滥用可封号并配合司法调查

## 备选方案

| 方案 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 强制实名 + 应用审核 | 滥用风险最低 | 接入门槛高、运维成本高 | Arm Pro 也没做,过度设计 |
| 完全开源无约束 | 最大化传播 | 滥用风险极高 | 与项目定位冲突 |
| 协议约束(本方案) | 平衡门槛与合规 | 滥用追溯依赖法律 | 行业惯例 |

## 影响

- 正面:接入门槛低,开发者愿意使用
- 负面:仍可能被灰产滥用,需保留追溯能力(注入水印等)
- 风险:被滥用为灰产工具链时,需保留法律追责依据
