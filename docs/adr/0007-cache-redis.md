# ADR 0007 · 缓存:Redis 单机起步

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:技术栈

## 背景

卡密验证是高频读,必须缓存。同时需要 nonce 防重放、限流计数、分布式锁。

## 决策

### 缓存:Redis 7
- **MVP 阶段**:单机模式
- **SaaS 上量后**:升级 Redis Cluster(预设升级路径)
- **应用层**:NestJS 用 `cache-manager` + `cache-manager-redis-yet`

### 缓存内容
- 卡密验证结果(TTL 60s,减少 DB 压力)
- nonce 防重放(TTL 5min)
- 限流计数器(TTL 1min/1hour)
- 设备指纹 -> 卡密绑定关系(TTL 1hour)
- 开发者会话(TTL 7day)

### 关键约束
- 不缓存敏感数据明文(卡密明文绝不进 Redis)
- 所有缓存 key 必须含 `tenant_id` 前缀,防跨租户泄露
- 缓存失效采用 LRU + TTL 双策略

### 升级路径
单机 -> 主从 -> 哨兵 -> Cluster。配置走环境变量,切换不改代码。

## 备选方案

| 方案 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 无缓存 | 简单 | DB 压力大 | 不可行 |
| 本地缓存(Caffeine) | 快 | 不支持分布式 | 多实例不一致 |
| Redis 单机(本方案) | 简单、够用 | 单点风险 | MVP 阶段可接受 |
| Redis Cluster | 高可用 | 运维复杂 | MVP 过度设计 |

## 影响

- 正面:MVP 阶段简单够用,QPS < 1000 无压力
- 负面:单点故障会导致验证失败,需有降级策略(失败时直接查 DB)
- 风险:Redis 重启会丢失 nonce 缓存,可能被重放攻击,需结合时间戳校验
