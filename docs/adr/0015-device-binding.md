# ADR 0015 · 设备绑定策略

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:功能

## 背景

设备绑定决定卡密能否跨设备使用,直接影响盗版传播风险与用户体验。

## 决策

### 绑定策略:开发者可配,默认首次激活绑定
| 策略 | 说明 | 默认 |
|---|---|---|
| (a) 不绑定 | 任意设备可用 | ❌ |
| (b) 首次激活绑定 | 首次使用锁定设备 | ✅ 默认 |
| (c) 一卡 N 机 | 限制最多 N 台设备(N 上限 5) | 可选 |

### 配置方式
- 卡密生成时选择绑定策略
- 同一应用下不同卡密可有不同策略
- 策略写入卡密属性,激活后不可改

### 解绑机制
- **开发者后台手动解绑**:用户换机时,开发者可在后台解除设备绑定
- **不可用户自助解绑**:否则绑定无意义
- 解绑后卡密可重新激活到新设备
- 解绑操作记录审计日志

### 一卡 N 机限制
- N 值由开发者配置,平台上限 5(防滥用)
- 同一卡密激活第 N+1 台设备时拒绝
- 已激活设备列表可在后台查看

### 不绑定的风险
- 工具类应用可接受(开发者选择)
- 但卡密一旦泄露,任意设备可用
- 建议开发者用"不绑定 + 限流"组合

## 备选方案

| 策略 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 强制一机一码 | 最严 | 用户换机麻烦 | 不灵活 |
| 默认不绑定 | 体验好 | 易盗版 | 不安全 |
| 开发者可配(本方案) | 灵活 | - | 合理 |

## 影响

- 正面:开发者按应用场景选择,平衡安全与体验
- 负面:解绑操作增加开发者支持成本
- 风险:开发者后台解绑被滥用(开发者自己帮用户解绑刷量),需结合 D3 风控
