# ADR 0017 · 离线验证策略

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:功能

## 背景

完全在线验证体验差(地铁/弱网),永久离线易破解。需平衡。

## 决策

### 离线策略:缓存 N 天,超期禁用
- **默认缓存 7 天**
- **开发者可配 1-30 天**
- 首次必须联网激活
- 之后每次启动联网验证,但允许缓存 N 天
- 超过 N 天未联网则禁用

### 缓存内容
- 卡密 hash
- 设备绑定状态
- 剩余有效时间
- 上次验证时间
- 服务端下发的 `cacheKey`(用于派生 AES 密钥)

### 缓存加密(详见 ADR 0026)
- AES-256-GCM 加密
- 密钥由 `cacheKey + deviceFingerprint` 在 Rust so 内派生
- 缓存带 HMAC 签名,篡改后下次验证直接拒绝
- 存应用内部存储 `/data/data/<pkg>/files/`,权限 0700

### 服务端下发指令
- **强制下线**:下次联网时生效,用于封禁已泄露卡密
- **延长缓存**:服务端可临时延长缓存期(如节假日网络预期拥塞)

### 工作流
```
启动 APP
  └─> 读取本地缓存
      ├─> 缓存有效且 < N 天:本地验证通过,允许使用
      └─> 缓存过期或无缓存:
          ├─> 联网验证
          │   ├─> 成功:更新缓存,允许使用
          │   └─> 失败:拒绝使用,提示用户
          └─> 无网络:按缓存策略
              ├─> 缓存 < N 天:允许使用
              └─> 缓存 > N 天:拒绝使用
```

## 备选方案

| 方案 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 完全在线 | 最安全 | 体验差 | 用户骂 |
| 一次性激活 + 永久离线 | 体验好 | 易破解 | 不安全 |
| 缓存 N 天(本方案) | 平衡 | - | 合理 |
| 缓存 + 心跳 | 防长期离线 | 心跳可被 patch | 复杂 |

## 影响

- 正面:用户出差/断网 7 天可接受,破解者需每 7 天重新激活
- 负面:开发者需处理"超期禁用"用户投诉
- 风险:缓存被 patch 改时间,需 HMAC 签名 + 服务端时间戳校验
