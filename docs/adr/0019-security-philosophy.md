# ADR 0019 · 安全设计哲学

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:安全

## 背景

小城笺是开源 + SaaS 双模式,客户端代码完全公开。必须承认根本约束,确立可执行的安全哲学。

## 决策

### 安全边界前置声明
**开源 + SaaS 双模式下的根本事实:**

| 事实 | 后果 |
|---|---|
| 客户端代码完全公开 | 攻击者 100% 知道算法 |
| 客户端在攻击者手里 | 任何 patch 都可能 |
| Rust so 也会被逆 | 只是难度高,不是不可能 |
| 服务端会被 CC/枚举 | 必须有风控 |
| 开源版自部署服务端不归项目管 | SaaS 安全策略不能依赖服务端独有配置 |

### 安全哲学
**不追求"防住所有攻击",而是"提高破解成本 + 检测异常 + 快速响应"**

三层目标:
1. **提高破解成本**:Rust so + 反调试 + 完整性校验,让攻击者投入大量时间
2. **检测异常**:服务端风控检测枚举/异常设备/异常 IP
3. **快速响应**:封禁卡密 / 强制下线 / 拉黑设备

### 不追求的目标
- ❌ "无法破解":开源下不可能
- ❌ "永久防护":任何防护都有破解时
- ❌ "防住所有 patch":客户端在攻击者手里

### 追求的目标
- ✅ 破解成本 > 破解收益(让破解者觉得不值)
- ✅ 异常检测覆盖率 > 90%
- ✅ 封禁响应时间 < 1 小时

## 备选方案

| 哲学 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 绝对防护 | 安全 | 不可能实现 | 开源下不可行 |
| 不做防护 | 简单 | 盗版横行 | 与项目目标冲突 |
| 提高成本 + 检测 + 响应(本方案) | 务实 | 需持续投入 | 合理 |

## 影响

- 正面:务实的安全目标,避免过度投入无法实现的目标
- 负面:仍有破解风险,需持续迭代
- 风险:开发者可能误解为"小城笺无法破解",需文档说明
