# ADR 0022 · 服务端防爆破

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:安全

## 背景

开源后卡密格式公开,攻击者会暴力枚举卡密。这是开源后最大的服务端风险。

## 决策

### 防爆破组合方案

| 机制 | 说明 |
|---|---|
| (a) IP 限流 | 同 IP 每分钟 N 次(默认 60) |
| (b) 设备指纹限流 | 同机器码每分钟 N 次(默认 30) |
| (c) 失败次数锁定 | 同设备连续失败 5 次锁 1 小时;同 IP 连续失败 20 次锁 1 小时 |
| (e) 卡密错误延迟 | 第 1 次失败 0.5s,第 2 次 1s,第 3 次 2s,指数退避 |

### 不做风控引擎(d)
- 多维度行为分析放 v2
- MVP 先用规则

### 限流配置
- **开发者可调**(不同应用承受能力不同)
- **平台级下限**:IP 60/min、设备 30/min(开发者不可低于此值)
- 配置存应用级别,后台可改

### Luhn 校验位前置
- C3 选了 Luhn 校验位
- "完全错误的卡密"在客户端就被拒,省一次服务端请求
- 服务端只接收格式合法的卡密

### 失败响应策略
- **不区分错误原因**:统一返回 `CARD_INVALID`,不告诉攻击者是格式错还是已被激活
- **延迟统一**:无论卡密是否存在,失败响应都延迟相同时间(防时序攻击)

### 风控日志
- 所有失败请求记录安全日志
- 字段:`IP / machineId / cardKeyHash / timestamp / failReason`
- 卡密明文绝不进日志
- 触发限流时告警飞书机器人

## 备选方案

| 方案 | 强度 | 不选原因 |
|---|---|---|
| 仅 IP 限流 | 低 | 攻击者换 IP 绕过 |
| IP + 设备指纹 | 中 | 无失败锁定 |
| IP + 设备 + 失败锁定 + 错误延迟(本方案) | 高 | 合理 |
| 风控引擎 | 最高 | MVP 过度设计 |

## 影响

- 正面:暴力枚举在 10 次后慢到不可用
- 负面:正常用户输错卡密也会被锁,需开发者后台解绑
- 风险:开发者自己测试时可能触发锁定,需提供测试模式白名单
