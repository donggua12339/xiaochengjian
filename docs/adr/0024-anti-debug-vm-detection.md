# ADR 0024 · 反调试与 VM 检测

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:安全

## 背景

被调试 = 被逆向,必须检测。模拟器被用于批量刷卡密,需检测。

## 决策

### MVP 检测项

| 检测项 | 实现方式 | 误报率 |
|---|---|---|
| `ptrace` 自附加 | Rust 调 `ptrace(PTRACE_TRACEME)` | 低 |
| `/proc/self/status` TracerPid | 读 TracerPid 字段 | 低 |
| Frida 检测 | 扫描 27042 端口 + frida-gadget 特征文件 | 中 |
| 模拟器检测 | build.prop 关键字段 + 传感器特征 | 中 |

### 不做 Root 检测
- 大量正常用户 root 自己设备(开发者、极客),误报会导致用户流失
- Magisk Hide 能轻松绕过
- 由开发者按需自行添加(不强制)

### 不做 Xposed 检测
- Xposed 已式微,LSPosed 兼容性差
- 误报率高

### 检测策略
- **分散检测**:检测逻辑分散在多个 Rust 函数中,结果**累积**判断
- **延迟响应**:检测到异常**不是直接崩溃**,而是:
  1. 标记异常状态
  2. 上报服务端(异步)
  3. 下次验证时拒绝(不立即拒绝)
- **目的**:增加攻击者定位难度

### 模拟器检测配置
- **开发者可配**:是否拒绝模拟器
- 部分应用确实需要模拟器测试
- 默认拒绝,开发者可关闭

### 检测实现位置
- 所有检测在 **Rust so 内**(防 patch)
- 不依赖 Kotlin 层检测(易绕过)

## 备选方案

| 方案 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 全检测(含 Root/Xposed) | 最严 | 误报高 | 用户流失 |
| MVP 精简检测(本方案) | 平衡 | - | 合理 |
| 不检测 | 简单 | 易逆向 | 不安全 |

## 影响

- 正面:基础反逆向,提升破解成本
- 负面:模拟器测试需开发者关闭检测
- 风险:误报导致正常用户被拒,需提供"申诉"通道(开发者后台解绑)
