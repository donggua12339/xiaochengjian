# ADR 0032 · 监控告警与日志

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:运维

## 背景

SaaS 版需要业务/系统/安全三层监控,开源版需要轻量日志方案。

## 决策

### 监控告警:Prometheus + Grafana + AlertManager

### 业务指标
- 验证 QPS
- 失败率
- 卡密激活数
- 异常设备数
- 限流触发数

### 系统指标
- CPU / 内存 / 磁盘 / 网络
- PG 连接数 / 慢查询
- Redis 内存 / 命中率

### 安全指标
- 失败登录数
- 异常 IP
- 卡密枚举告警(同 IP 1 分钟失败 > 20)
- 注入工具调用量

### 告警通道
- **飞书机器人**(MVP)
- QQ 机器人(v2)
- 紧急告警:飞书电话

### 关键告警阈值
- 验证失败率 > 5%
- PG 连接数 > 80%
- 磁盘 > 85%
- 卡密枚举告警(同 IP 1 分钟失败 > 20)
- Redis 内存 > 80%

### 日志系统:Loki + Promtail

| 日志类型 | 内容 | 保留期 |
|---|---|---|
| 应用日志 | NestJS 结构化 JSON | 30 天 |
| 访问日志 | Nginx access log | 7 天 |
| 审计日志 | 开发者后台操作 | 1 年 |
| 安全日志 | 失败验证、限流、注入调用 | 1 年 |
| 卡密敏感数据 | ❌ 不记明文,只记 hash 前 4 位 | - |

### 日志格式
- 结构化 JSON
- 关键字段:`timestamp / level / requestId / tenantId / appId / event / meta`
- 强制 `cardKey` / `password` / `secret` 字段输出 `***`

### 审计日志单独存 PG
- 不进 Loki,保证可查询可导出
- 表 `audit_log`:开发者 ID / 操作 / 时间 / IP / 详情
- 保留 1 年

### 开源版简化
- Loki 可选,默认禁用
- 应用日志写本地文件 + stdout
- 审计日志强制存 PG

## 备选方案

| 方案 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| ELK | 全功能 | 太重 | 资源占用高 |
| Loki + Promtail(本方案) | 轻量 | 索引弱 | 合理 |
| 仅文件日志 | 简单 | 不可查询 | SaaS 不够用 |

## 影响

- 正面:开源版轻量,SaaS 版全功能
- 负面:开源版日志查询能力弱
- 风险:Loki 单点故障,需备份
