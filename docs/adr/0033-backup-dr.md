# ADR 0033 · 数据备份与灾备

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:运维

## 背景

卡密数据是开发者命脉,丢失 = 用户卡密作废 = 集体投诉。需严格备份与灾备方案。

## 决策

### 数据备份方案

| 项 | 方案 |
|---|---|
| PG 备份频率 | 全量每日 1 次 + WAL 增量实时 |
| 备份存储 | 本地 + 异地(对象存储 OSS/COS) |
| 备份加密 | AES-256,密钥与数据库分离 |
| 保留期 | 7 天滚动 + 4 周末备份 + 12 月末备份 |
| RPO | 5 分钟(WAL 流复制) |
| RTO | 30 分钟 |
| 灾备演练 | 每季度 1 次恢复演练 |

### 备份加密
- 备份文件 AES-256 加密
- 加密密钥**不在数据库服务器**,单独 KMS 管理
- 开源版用本地文件 + chmod 600
- SaaS 版用云厂商 KMS

### 备份存储
- **本地**:服务器本地磁盘,快速恢复
- **异地**:对象存储(阿里云 OSS / 腾讯云 COS),< 10 元/月
- 异地备份加密后上传,即使对象存储泄露也无法解密

### 恢复演练
- 每季度 1 次
- 从备份恢复到测试环境,验证数据完整性
- 记录恢复时间,验证 RTO

### 开源版 vs SaaS 版

| 项 | 开源版 | SaaS 版 |
|---|---|---|
| PG 备份 | `pg_dump` 脚本 + cron | WAL 流复制 + 异地 |
| 异地备份 | 可选 | 强制 |
| 加密 | 推荐 | 强制 |
| 演练 | 文档指导 | 每季度执行 |

### 不备份的内容
- **卡密明文**:服务端不存明文,无需备份
- **缓存数据**:Redis 数据可重建,不备份
- **日志**:Loki 日志不备份(短期数据)

## 备选方案

| 方案 | RPO | 成本 | 不选原因 |
|---|---|---|---|
| 仅每日全量 | 24h | 低 | 数据丢失多 |
| WAL 流复制(本方案) | 5min | 中 | 合理 |
| 实时双写 | 0s | 高 | 过度设计 |

## 影响

- 正面:数据丢失风险极低
- 负面:备份存储成本(可接受)
- 风险:加密密钥丢失导致备份不可用,需密钥备份方案
