# ADR 0035 · 版本发布与 i18n

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:运维

## 背景

版本发布策略影响开发者集成体验。i18n 决定开发工作量。

## 决策

### 版本发布策略

| 项 | 方案 |
|---|---|
| 版本号 | 语义化版本 `MAJOR.MINOR.PATCH` |
| 发布频率 | PATCH 按需,MINOR 双周,MAJOR 按年 |
| 预发布 | `alpha` -> `beta` -> `rc` -> 正式 |
| 灰度 | SaaS 版按租户分批(5% -> 20% -> 100%) |
| 回滚 | 镜像保留最近 10 个版本,一键回滚 |
| LTS | 每 6 个月一个 LTS,维护 1 年 |

### MVP 阶段不灰度
- 用户少,灰度无意义
- 用户数 > 100 后开始灰度

### SDK 与后端版本独立
- SDK 每月发
- 后端每周发
- 注入工具与 SDK 版本同步(ADR 0037)

### LTS 必要
- 开发者集成 SDK 后不希望频繁升级
- LTS 给稳定承诺

### 回滚必须快
- K8s `kubectl rollout undo` 一键,5 分钟内
- Compose 用旧镜像 tag 重启

### 破坏性变更
- 必须 MAJOR 版本
- 提供迁移指南
- 旧版本维护 6 个月过渡期

### i18n:中文为主预留接口

| 项 | MVP | v2 |
|---|---|---|
| 后台界面 | 中文 | 中英双语 |
| 文档 | 中文 | 中英双语 |
| 错误信息 | 错误码英文(`CARD_NOT_FOUND`) | 同 |
| 邮件模板 | 中文 | 双语 |
| 协议文本 | 中文 | 双语 |

### 代码层预留 i18n
- 用 `vue-i18n` + `nestjs-i18n`
- 文案全部走 key,不硬编码
- 未来加语言只加翻译文件

### 不选中英双语的原因
- 双语维护成本翻倍
- MVP 工期 +30%
- 目标用户是中国开发者

### SDK 错误码
- 必须英文(如 `CARD_NOT_FOUND`、`DEVICE_LOCKED`)
- 不本地化,方便开发者查日志
- 错误码列表公开文档

## 备选方案

| 方案 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 仅中文 | 简单 | 国际化受限 | 不可扩展 |
| 中英双语 | 国际化 | 工期 +30% | MVP 过重 |
| 中文为主预留 i18n(本方案) | 平衡 | - | 合理 |

## 影响

- 正面:MVP 工期可控,未来可扩展
- 负面:海外用户需等 v2
- 风险:文案硬编码漏网,需 lint 检查
