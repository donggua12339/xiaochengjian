# ADR 0038 · 测试策略

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:工程规范

## 背景

卡密验证系统是安全敏感项目,测试覆盖率直接影响安全性。

## 决策

### 测试覆盖率目标

| 层 | 工具 | 覆盖率 | 卡点 |
|---|---|---|---|
| Rust 核心安全模块 | `cargo test` + `cargo-tarpaulin` | ≥ 90% | CI 卡 |
| NestJS 后端 | Jest | ≥ 80% | CI 卡 |
| 集成测试关键路径 | Testcontainers(PG + Redis) | 100% | CI 卡 |
| Android SDK | JUnit + Robolectric | ≥ 70% | 警告 |
| Vue 后台 | Vitest + Vue Test Utils | ≥ 60% | 警告 |
| E2E | Playwright(后台) | 登录/卡密生成/激活主流程 | v2 |
| 注入工具 | 黄金 APK 测试集 | 10 个真实 APK 注入不崩溃 | CI 卡 |

### MVP 阶段聚焦
- **Rust 90%**:安全核心必须高
- **NestJS 80%**:业务逻辑必须有测试
- **集成测试关键路径**:用 Testcontainers 起真实 PG + Redis
- **E2E 放 v2**:MVP 阶段手动测

### 集成测试关键路径
- 开发者注册 -> 2FA -> 创建应用
- 生成卡密 -> 导出 CSV
- SDK 激活卡密 -> 设备绑定
- SDK 验证卡密 -> 缓存命中
- 卡密禁用 -> 下次验证拒绝
- 设备解绑 -> 重新激活

### Rust 核心测试
- 关键算法必须有**黄金向量测试**
- 机器码算法:固定输入 -> 固定输出
- 加解密:已知向量
- 签名:RFC 测试向量
- 不测 JNI 桥接层(用 Kotlin 集成测试覆盖)

### 注入工具黄金 APK 测试集
- 收集 10 个不同混淆/包结构/签名方案的 APK
- CI 每次跑注入不崩溃
- 覆盖:无混淆 / ProGuard / R8 / 加固(壳 dex) / multidex / AndroidX / 老版本

### 测试数据管理
- 测试数据库用 Testcontainers 临时容器
- 不依赖外部数据
- 每次测试隔离

### 性能测试
- 放 v2
- MVP 阶段用 `autocannon` 手动测验证 API QPS

### 安全测试
- 放 v2
- MVP 阶段对照 ADR 0019-0027 安全决策手动 review

## 备选方案

| 方案 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 全层 90%+ | 质量最高 | 工期翻倍 | 过度 |
| MVP 聚焦核心(本方案) | 平衡 | UI 测试弱 | 合理 |
| 仅手动测试 | 快 | 不可靠 | 不可接受 |

## 影响

- 正面:核心安全有保障
- 负面:UI 层 bug 可能漏网
- 风险:Rust 90% 卡点可能导致开发速度慢
