# ADR 0039 · 文档规范

- 状态:accepted
- 日期:2026-07-13
- 决策者:小城笺项目
- 层次:工程规范

## 背景

开源项目文档质量直接影响开发者接入意愿。需明确文档类型与规范。

## 决策

### 文档类型

| 文档类型 | 内容 | 位置 |
|---|---|---|
| README | 项目简介、快速开始、合规红线 | 仓库根 |
| ARCHITECTURE | 架构图、模块依赖、技术决策 | `docs/architecture.md` |
| API 文档 | 后端 REST API + SDK API | OpenAPI 3.0 自动生成 |
| SDK 集成指南 | Gradle 引入、初始化、调用 | `docs/sdk-guide.md` |
| 注入工具指南 | 使用说明、免责协议 | `docs/injector-guide.md` |
| 部署指南 | Compose / K8s 部署 | `docs/deploy.md` |
| 安全白皮书 | 五层防护、威胁模型 | `docs/security.md` |
| ADR | 每个重大决策 + 理由 | `docs/adr/` |
| CLA | 贡献者协议 | `docs/cla.md` |
| 用户协议 | 合规红线、免责条款 | `docs/compliance/user-agreement.md` |

### ADR 规范
- 使用 `docs/adr/0000-template.md` 模板
- 编号连续,不重用
- 决策变更写新 ADR,标记旧 ADR 为 `superseded by NNNN`
- 每个 ADR 含:背景 / 决策 / 备选方案 / 影响 / 关联

### API 文档
- NestJS 用 `@nestjs/swagger` 自动生成 OpenAPI 3.0
- 前端用 `openapi-typescript` 生成 TypeScript 客户端
- 避免手写漂移

### SDK 集成指南
- **必须含可复制粘贴的代码**
- 开发者最讨厌"看完文档不会用"
- 含完整 demo 项目链接

### 安全白皮书
- **公开**(开源项目透明化安全设计,反而提升信任)
- 五层防护详解
- 威胁模型
- 已知限制
- 不公开的内容:具体混淆算法、密钥派生细节

### 合规红线
- 写进 README 第一屏
- 用户协议明确法律条款

### 文档与代码同步
- 文档与代码同仓库
- PR 改代码必须改文档
- CI 检查文档覆盖

### 文档语言
- 中文为主(ADR 0035)
- 错误码英文
- v2 加英文翻译

### 文档版本
- 文档头部标注最后更新日期
- 重大变更在文档顶部加 changelog

## 备选方案

| 方案 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 最简文档 | 省时 | 接入难 | 不可接受 |
| 全文档(本方案) | 接入友好 | 维护成本 | 合理 |
| 仅 README | 简单 | 不够 | 不可接受 |

## 影响

- 正面:开发者接入体验好,社区信任
- 负面:文档维护成本高
- 风险:文档与代码漂移,需 CI 检查
