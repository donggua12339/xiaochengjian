# ADR 0040 · 开源发布时机:SaaS 上线后开源

- 状态:accepted
- 日期:2026-07-14
- 决策者:小城笺项目
- 层次:战略

## 背景

MVP(M0-M3)完成后,需要决定开源与 SaaS 上线的先后顺序。开源引流社区,SaaS 商业化。

## 决策

**SaaS 上线后开源**

先完成 SaaS 上线(支付/服务器/监控),跑稳后再开源。

## 备选方案

| 方案 | 优点 | 缺点 | 不选原因 |
|---|---|---|---|
| 立即开源 | 社区引流早 | 代码刚成型,可能有 bug | 质量风险 |
| SaaS 上线后开源(本方案) | 质量稳 | 错过早期社区 | 合理 |
| 开源 + SaaS 同步 | 双线并行 | 压力大 | 资源不足 |
| 开源核心 + SaaS 闭源增值 | 商业保护 | 失去开源意义 | 与 ADR 0002 矛盾 |

## 影响

- 正面:SaaS 上线验证商业模式后,开源版质量有保障
- 负面:开源推迟,社区冷启动慢
- 风险:SaaS 上线周期长会推迟开源时机

## 关联

- 关联 ADR:0002(双模式)、0041(SaaS 边界)
- 执行顺序:SaaS 上线 -> 开源
