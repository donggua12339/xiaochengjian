# ADR 0041 · SaaS 与开源版功能边界:SaaS 增值

- 状态:accepted
- 日期:2026-07-14
- 层次:战略

## 背景

ADR 0002 定了"开源 + SaaS 双模式",但功能边界未明确。

## 决策

**SaaS 增值模式**

- **开源版**:核心功能(验证/卡密/应用/2FA/SDK/注入工具 CLI),功能完整可用
- **SaaS 独占**:多租户运维、Prometheus/Grafana 监控、AlertManager 飞书告警、Web APK 注入工具、SLA

## 备选方案

| 方案 | 开源版 | SaaS 独占 | 不选原因 |
|---|---|---|---|
| 完全相同 | 全功能 | 无 | SaaS 无增值点 |
| SaaS 增值(本方案) | 核心 | 运维/监控/Web 注入 | 合理 |
| SaaS 独占管理后台 | 仅 SDK+API | 管理后台闭源 | 开源版无法管理卡密 |

## 影响

- 正面:开源版不阉割核心,社区信任;SaaS 增值清晰
- 负面:开源版用户自部署需自己搭监控
- 关键:SaaS 独占功能用 feature flag(`ENABLE_SAAS`)控制,同仓库
