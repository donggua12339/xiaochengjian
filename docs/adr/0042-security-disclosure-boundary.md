# ADR 0042 · 安全设计公开边界:架构公开,实现保留

- 状态:accepted
- 日期:2026-07-14
- 层次:安全

## 背景

开源后安全设计面临"公开多少"的抉择。全公开给攻击者指南,全闭源失去开发者信任。

## 决策

**架构公开,实现保留**

- **公开**:五层防护架构、威胁模型、ADR、安全白皮书、Rust so 源码
- **保留**(文档不写具体值):混淆参数、自校验 hash 值、密钥派生盐值、反调试延迟策略具体参数

## 备选方案

| 方案 | 公开 | 保留 | 不选原因 |
|---|---|---|---|
| 全公开 | 所有 | 无 | 等于破解指南 |
| 架构公开,实现保留(本方案) | 架构+源码 | 具体参数 | 合理 |
| 仅 API 契约 | API | 所有实现 | 失去开发者信任 |

## 影响

- 正面:开发者信任(知道防护深度),破解成本高(不知道具体参数)
- 负面:Rust 源码开源后,攻击者能逆向实现
- 关键:安全白皮书(ADR 0039)公开架构,不写"自校验 hash = 0xABC123"
