# ADR 0054 · README 深度:标准 + docs/ 链接

- 状态:accepted
- 日期:2026-07-14
- 层次:文档

## 背景

README 是开源仓库第一印象,需定深度。

## 决策

**标准深度 + docs/ 链接**

README 内容:
1. 项目定位(1 段)
2. 5 层安全防护图(架构图)
3. 3 分钟快速开始(docker compose up)
4. 功能特性列表
5. 链接到 docs/(ADR/安全白皮书/部署指南)
6. Badge:CI 状态 / License / 版本 / 贡献者

## 备选方案

| 方案 | 深度 | 不选原因 |
|---|---|---|
| 最简(5 分钟) | 简介+开始+License | 显得不专业 |
| 标准(本方案) | +架构+部署+FAQ | 合理 |
| 详尽(1 小时) | +ADR+白皮书+路线图 | README 太长 |

## 影响

- 正面:20 分钟读完,专业度够
- 负面:详尽内容在 docs/,多一次跳转
- 关键:加 Badge 提升信任(CI 绿 / License 明确)

## 关联

- 关联 ADR:0039(文档规范)、0053(仓库结构)
