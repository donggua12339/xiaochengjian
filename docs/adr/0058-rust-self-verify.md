# ADR 0058 · Rust so 自校验:编译时嵌入 + 服务端下发

- 状态:**superseded by 0069**(v2 撤除 so 自校验,服务端验证是权威)
- 日期:2026-07-14
- 层次:安全

> **本 ADR 已被 [ADR 0069](0069-rust-self-verify-removal.md) 取代**(2026-07-19)。
> v2 重构撤掉了 .text 段 hash 嵌入与服务端 so hash 下发,理由:开源 + 服务端验证权威 + 阻碍合法审计。
> 本文档保留作为决策追溯,不再有效。

## 背景

ADR 0025 要求 so 自校验,防被 patch。当前未实现。

## 决策

**编译时嵌入 .text hash + 服务端下发预期 hash 组合**

- 编译时:build.rs 读取 .text 段计算 hash,嵌入常量(obfstr 混淆)
- 运行时:JNI 入口(native01-08)各自校验不同段
- 服务端:开发者后台配"预期 hash",SDK 启动拉取对比

## 备选方案

| 方案 | 校验 | 不选原因 |
|---|---|---|
| 编译时嵌入 | 单次 | hash 常量可 patch |
| 分散校验 | 多点 | 无服务端兜底 |
| 服务端下发 | 动态 | 无编译时兜底 |
| 组合(本方案) | 双重 | 合理 |

## 影响

- 正面:双重校验,patch 难度最高
- 负面:服务端下发增加网络请求
- 开发量:2 天(build.rs + 分散校验点 + 服务端接口)
