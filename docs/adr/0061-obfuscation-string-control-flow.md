# ADR 0061 · 混淆策略:字符串 + 控制流

- 状态:accepted
- 日期:2026-07-14
- 层次:安全

## 背景

ADR 0023 提到 obfstr 混淆,当前未用。用户要求"对安全性很重视"。

## 决策

**字符串混淆(obfstr)+ 控制流混淆(olvllvm-rust),虚拟化作为长期目标**

- 字符串混淆:obfstr(已有依赖),密钥派生盐值/RSA 公钥指纹/错误信息加密
- 控制流混淆:olvllvm-rust(控制流平坦化 + 虚假控制流)
- 虚拟化:长期目标(SaaS 版独占,开源版无)

## 备选方案

| 方案 | 混淆 | 不选原因 |
|---|---|---|
| 无混淆 | 源码可读 | 不安全 |
| 字符串(本方案基础) | obfstr | 不够 |
| 字符串 + 控制流(本方案) | + olvllvm | 合理(开源极限) |
| 全套(+ 虚拟化) | + VM | 开源后效果打折 + 数月开发 |

## 影响

- 正面:开源项目安全极限(字符串 + 控制流)
- 负面:olvllvm-rust 不成熟,可能需自写 pass(2 周)
- 性能:控制流平坦化 10-30% 开销
- 虚拟化:SaaS 版用商业 VM(闭源),开源版无
