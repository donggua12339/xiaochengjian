# ADR 0062 · 防重打包:直接读 APK + 服务端下发

- 状态:accepted
- 日期:2026-07-14
- 层次:安全

## 背景

M2.6 的 APK 签名校验调用系统 API,可被 hook 绕过。

## 决策

**直接读 APK 文件解析签名 + 服务端下发预期 hash**

- 客户端:直接读 APK 文件解析签名(integrity.rs 已实现 compute_apk_signature_hash)
- 服务端:开发者后台配"预期签名 hash",SDK 启动拉取对比
- 不一致 -> 拒绝验证

## 备选方案

| 方案 | 校验 | 不选原因 |
|---|---|---|
| 系统 API | 低(hook 绕过) | 不安全 |
| 直接读 APK(本方案基础) | 中 | 无服务端兜底 |
| 直接读 + 服务端下发(本方案) | 高 | 合理 |
| + 多点校验 | 最高 | 性能开销 |

## 影响

- 正面:不依赖系统 API,服务端兜底
- 负面:服务端下发增加网络请求
- 攻击者:重打包后签名变,客户端对比服务端值不一致 -> 拒绝
- 攻击者改客户端跳过:Rust so 自校验(ADR 0058)兜底
- 开发量:1 天

## 关联

- 关联 ADR:0025(完整性校验)、0058(so 自校验)
