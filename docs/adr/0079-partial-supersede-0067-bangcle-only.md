# ADR 0079 · 部分取代 ADR 0067(仅梆梆自检场景)

- 状态:proposed(与 ADR 0078 同步,律师前置)
- 日期:2026-07-20
- 决策者:小城笺项目
- 层次:功能 / 合规

## 背景

ADR 0067(2026-07-14 accepted)定调:MVP 不支持加固 APK,壳 dex 注入 + 自动脱壳延后到 v3。该决策在 v2 阶段仍有效。

ADR 0077(2026-07-19 accepted)引入"自有 APK 诊断",其例外 B(2026-07-20 修订)允许对自有梆梆加固 APK 做加固层自检(完整性报告,不脱壳不反编译),具体实现见 ADR 0078。

ADR 0078 的功能与 ADR 0067 的"MVP 不支持加固 APK"产生部分冲突:对梆梆这一家厂商的自有 APK,允许做有限自检;对其他加固厂商,仍按 ADR 0067 拒绝。

需明确:
1. ADR 0078 与 ADR 0067 的边界(哪些被取代,哪些不变)
2. 取代的前提条件(律师意见、3 把锁约束)

## 决策

**ADR 0079 部分取代 ADR 0067,范围仅限"梆梆加固自有 APK 的加固层自检"。**

### 取代范围(梆梆场景下,以下行为从"拒绝"变为"允许")

| ADR 0067 原决策 | ADR 0079 取代后(梆梆场景) | 取代条件 |
|---|---|---|
| 检测到加固 -> 拒绝注入,提示"请提供未加固 APK" | 检测到**梆梆**加固 -> 允许做加固层自检(完整性报告,不脱壳) | 通过 ADR 0077 三重校验 + ADR 0078 锁 A/B/C + 律师意见 |
| 自动脱壳延后 v3 | 仍延后 v3,**不脱壳** | 本 ADR 不取代该条 |
| 壳 dex 注入延后 v2 | 仍延后,**不做注入** | 本 ADR 不取代该条 |

### 不取代范围(以下行为 ADR 0067 仍有效)

1. **其他加固厂商**(360 / 爱加密 / 乐固 / 腾讯乐固 / 百度 / 通付盾 / 娜迦等):仍按 ADR 0067 拒绝,不做任何形式的自检或脱壳
2. **通用脱壳**:仍按 ADR 0067 延后 v3,本 ADR 不涉及
3. **壳 dex 注入**:仍按 ADR 0067 延后 v2,本 ADR 不涉及
4. **运行时 hook(LSPatch 类)**:仍按 ADR 0067 拒绝,本 ADR 不涉及
5. **梆梆加固 APK 的非自检操作**(注入 / 修改字节码 / 重打包等):仍按 ADR 0067 拒绝,本 ADR 不取代

### 取代的前提(任一不满足则 ADR 0067 仍完全有效)

1. **律师意见落地**:ADR 0078 §5 律师意见书通过
2. **ADR 0078 状态为 accepted**:从 proposed 改为 accepted
3. **3 把锁全部到位**:锁 A(仅梆梆)+ 锁 B(EULA)+ 锁 C(仅完整性报告)
4. **三重校验通过**:ADR 0077 §2 三重校验(包名白名单 + 签名 hash 比对 + 目录隔离)

**前提不满足时的回退**:本 ADR 自动失效,ADR 0067 恢复完全有效,梆梆适配器代码删除或保留但不启用(由用户决定)。

## 备选方案

| 方案 | 含义 | 不选原因 |
|---|---|---|
| A. 完全取代 0067 | 所有加固 APK 都允许自检 | 违反 ADR 0078 锁 A(仅梆梆),法律风险高 |
| B. 部分取代 0067(本方案) | 仅梆梆自检场景取代 | 用户决策 + 律师前置 |
| C. 不取代,新建独立 ADR 0080 | 不动 0067,加 0080 允许梆梆 | 决策关系混乱,违反 ADR 追溯规范(CLAUDE.md §10) |
| D. 修改 0067 原文 | 直接改 0067 内容 | 违反 CLAUDE.md §10(ADR 不可变,只能 superseded) |

## 影响

### 正面影响

- 明确 ADR 0078 与 0067 的边界,避免追溯混乱
- 限定梆梆场景的法律风险可控
- 前提条件回退机制提供安全网
- 与 ADR 0077 例外 B 形成完整合规链

### 负面影响

- ADR 0067 状态需更新为"部分 superseded by 0079"(不完全是 superseded,因其他加固方案仍按 0067)
- 取代前提复杂,需在代码中明确实现(检测到梆梆且 3 把锁全过才走 0078,否则走 0067)

### 风险

- **取代范围误判**:开发时把非梆梆加固 APK 也走了 0078 路径
  - 缓解:hardener-detector 严格匹配梆梆特征,非梆梆直接返回 `UNSUPPORTED_HARDENER`,与 ADR 0067 拒绝路径一致
- **律师意见被驳回**:ADR 0078 rejected,本 ADR 同步失效
  - 缓解:回退机制已定义,代码可一键禁用梆梆适配器
