# ADR 0082-B · 腾讯乐固自检适配器(V1.5a,优先实现)

- 状态:proposed(合规核查已通过,可开发)
- 日期:2026-07-21
- 决策者:小城笺项目
- 层次:功能 / 安全 / 合规
- 关联 ADR:0078(梆梆适配器,锁 A 扩展)

## 背景

ADR 0078 锁 A 原文:"仅梆梆一家,360 / 爱加密 / 乐固 / 腾讯乐固等其他加固方案明确不支持"。

V1.5 阶段扩展加固自检范围,腾讯乐固为国内独立开发者使用率 Top 3 之一。

本 ADR 扩展 ADR 0078 锁 A 的支持范围,增加腾讯乐固。

## 合规核查结论(已确认,2026-07-21)

### 核查依据

- 腾讯云移动应用安全(御安全)服务协议
- 腾讯乐固产品文档

### 逐条分析

腾讯乐固(御安全)的协议环境比 360 更宽松:

1. **协议未发现明确的"反向工程"禁令**:腾讯云服务协议主要限制"恶意攻击腾讯云服务""传播违法内容"等行为,未提及禁止对已加固 APK 进行完整性自检
2. **开发者权利保留**:开发者使用腾讯乐固加固后,APK 仍是开发者自有资产,腾讯未剥夺开发者对自有 APK 的安全运维权利
3. **与 ADR 0078 同构**:小城笺处理的是 `com.xxx.app`(开发者资产),而非 `com.tencent.*`(腾讯资产)

### 最终裁定

腾讯乐固协议**不构成**对 ADR 0082-B 的阻碍。合规风险已排除,可直接进入开发。

## 决策

### 腾讯乐固特征

| 特征 | 值 |
|---|---|
| so 文件 | libshell.so / libshella.so(腾讯乐固标志) |
| Application 类名前缀 | com.tencent.* |
| 加固产物 | lib/*/libshell.so + 原 dex 加密 |

### 锁 A 扩展(本 ADR)

ADR 0078 锁 A 从"仅梆梆"扩展为"梆梆 + 腾讯乐固"。其他锁(B/C + ADR 0081 锁 7)不变。

### 检测规则

```typescript
// 腾讯乐固 so 特征(从 UNSUPPORTED 列表移到支持列表)
const TENCENT_LEGU_SO_PATTERNS = [
  /^lib\/[^/]+\/libshell\.so$/i,
  /^lib\/[^/]+\/libshella\.so$/i,
];

const TENCENT_LEGU_APP_PREFIXES = ['com.tencent.'];
```

### EULA 前置(锁 B)

腾讯乐固独立 EULA 文本(`docs/compliance/audit-eula-legu.md`,待起草),开发者备案时强制勾选。

EULA 核心条款:"本人为 APK 著作权人,本次自检系对自有资产的安全审计"。

### 完整性报告(锁 C)

输出字段与梆梆适配器一致(soFiles / entryClass / signatures / suspiciousCalls / scanVersion / scanTime),不含源码。

### 优先级

V1.5a(**优先实现**),因为:
- 腾讯乐固 EULA 相对宽松(未发现明确反向工程禁令)
- 风险等级低于 360 加固保
- 快速补全 Top 3 覆盖

### 技术实现

- 复用 HardenerDetector 逻辑,检测 libshell.so 和 com.tencent.* 特征
- 适配器仅输出完整性报告(so hash、签名状态),严禁触碰 dex 或输出源码(维持锁 C)
- EULA 文本只需开发者确认:"本人为 APK 著作权人,本次自检系对自有资产的安全审计"
