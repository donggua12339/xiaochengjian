# 贡献者许可协议(CLA)

> 提交 PR 前请签署本协议。个人贡献者签署"个人 CLA",企业贡献者签署"企业 CLA"。

## 个人 CLA

### 贡献者声明

本人(以下称"贡献者")在向小城笺(以下称"项目")提交代码、文档或其他作品(以下称"贡献")时,声明如下:

1. **贡献来源合法**:贡献者保证所提交的贡献为本人原创,或本人有合法授权可提交,不侵犯任何第三方的著作权、专利权、商标权或其他知识产权。

2. **授权许可**:贡献者授予项目所有者(以下称"项目方")非独占的、全球范围内的、永久的、免费的、可转授权的许可,允许项目方以任何方式(包括但不限于复制、修改、发行、再许可、商用)使用贡献者的贡献。项目方可将贡献纳入 Apache 2.0 许可证或项目方自行选择的其他许可证下分发。

3. **专利授权**:贡献者授予项目方及其用户非独占的、全球范围内的、永久的、免费的专利许可,允许项目方及其用户使用贡献者贡献中涉及的专利。

4. **无保证**:贡献者保证其贡献在提交时不存在已知的缺陷,但不承诺贡献的功能性、可靠性或适用性。

5. **责任限制**:贡献者对贡献承担有限责任,不对项目方或第三方因贡献产生的间接损失负责。

6. **贡献者保留版权**:贡献者保留其贡献的版权,仅授予上述许可。

7. **配合司法**:若贡献内容涉及法律纠纷,贡献者同意配合调查。

### 签署信息

请复制以下内容填入 PR 描述(电子签署即生效):

```
CLA 个人签署
GitHub 用户名:[请填写]
真实姓名:[请填写]
邮箱:[请填写]
签署日期:[YYYY-MM-DD]
声明:本人已阅读并同意小城笺个人 CLA 全部条款。
```

## 企业 CLA(企业员工贡献时)

如贡献者受雇于企业,且贡献属于职务作品,需企业签署企业 CLA。模板请发邮件至 `cla@xiaochengjian.example` 索取。

## 常见问题

**Q: 为什么需要 CLA?**
A: 项目采用 Apache 2.0 + SaaS 双模式,需要确保所有贡献可被项目方商用,且未来许可证变更时无法律障碍。

**Q: CLA 是否意味着放弃版权?**
A: 不,贡献者保留版权,仅授予项目方许可。

**Q: 个人 PR 也要签吗?**
A: 是,所有 PR 均需签 CLA,即使是修复一个错别字。这是开源项目惯例。

**Q: CLA 电子签署是否具有法律效力?**
A: 根据《电子签名法》,电子签名与手写签名具有同等法律效力。在 PR 描述中填入签署信息即视为电子签署。
