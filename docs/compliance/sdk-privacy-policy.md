# xcj-auth-sdk 隐私政策

> 版本:1.0.0
> 生效日期:2026-07-21
> 依据:《个人信息保护法》+ 律师意见书 XCJ-2026-LC-0081

## 1. 概述

xcj-auth-sdk(小城笺卡密验证 SDK)在运行过程中会采集有限的设备与应用信息,用于卡密激活、设备绑定、鉴权验证等核心功能。本政策说明 SDK 采集的数据字段、用途、留存期限及共享规则。

## 2. 采集字段

| 字段 | 类型 | 用途 | 是否敏感 |
|---|---|---|---|
| app_id | String | 标识应用,关联开发者账号 | 否 |
| package_name | String | 校验 APK 包名(三重校验锁 1) | 否 |
| sign_hash | String | 校验 APK 签名(三重校验锁 1 + 锁 7 客户端自检) | 否 |
| OAID(匿名化) | String | 设备标识(匿名化,非 IMEI/Android ID) | 否(匿名化) |
| sdk_version | String | SDK 版本号(兼容性统计) | 否 |
| timestamp | Long | 请求时间戳(防重放) | 否 |

**不采集**:
- ❌ 通讯录
- ❌ 位置(GPS/Wi-Fi/基站)
- ❌ 短信 / 通话记录
- ❌ IMEI / Android ID / MAC 地址
- ❌ 应用列表
- ❌ 剪贴板
- ❌ 传感器数据

## 3. 数据用途

SDK 采集的数据仅用于:
1. **卡密激活**:验证卡密有效性,绑定设备
2. **鉴权验证**:校验应用包名 + 签名,防止二次打包
3. **设备绑定**:OAID 匿名化标识,限制卡密使用设备数
4. **防重放**:时间戳 + nonce,防止请求重放攻击
5. **兼容性统计**:SDK 版本号,用于兼容性分析(匿名聚合)

**不用于**:
- ❌ 用户画像 / 精准营销
- ❌ 广告推送
- ❌ 第三方数据共享
- ❌ 任何与卡密验证无关的用途

## 4. 数据留存

| 数据 | 留存期限 | 存储位置 |
|---|---|---|
| 激活记录 | 1 年(滚动清理) | 开发者服务器(小城笺 SaaS 或自部署) |
| 验证日志 | 1 年(滚动清理) | 开发者服务器 |
| 审计日志 | 1 年(ADR 0032) | 开发者服务器 |
| OAID 匿名化标识 | 随激活记录,1 年 | 开发者服务器 |

**不留存**:
- SDK 本地不持久化任何个人数据
- 所有数据仅传输到开发者配置的服务器(serverUrl)

## 5. 数据共享

- **不共享给第三方**:SDK 采集的数据仅传输到开发者配置的服务器,不共享给小城笺平台或其他第三方
- **开发者责任**:开发者作为数据控制者,需自行遵守《个人信息保护法》,在其 APP 隐私政策中披露 SDK 数据采集行为

## 6. 开发者义务

集成 xcj-auth-sdk 的开发者必须:
1. 在其 APP 隐私政策中披露 xcj-auth-sdk 的数据采集行为(本政策可作为参考)
2. 获取用户同意(如适用,依据《个人信息保护法》第 13 条)
3. 不得将 SDK 采集的数据用于本政策第 3 条以外的用途
4. 不得将数据共享给第三方(除非法律要求)

## 7. 用户权利

终端用户(开发者 APP 的使用者)有权:
1. 知情权:了解 SDK 采集的数据字段及用途
2. 删除权:要求开发者删除其激活记录(开发者服务器)
3. 撤回同意:卸载 APP 即视为撤回同意(SDK 本地不持久化数据)

## 8. 锁 6 数据锁定(ADR 0081)

xcj-auth-sdk 在 Packer 封装场景下,SDK 配置仅允许以下字段(锁 6 数据锁定):
- appId
- serverUrl
- offlineCacheDays
- oaidEnabled

**禁止配置**:
- ❌ contacts(通讯录)
- ❌ location(位置)
- ❌ sms(短信)
- ❌ callLog(通话记录)
- ❌ phone(电话)
- ❌ imei
- ❌ serialNumber
- ❌ macAddress

Packer 模块在校验 SDK 配置时,如检测到上述禁止字段,拒绝封装并返回 `DATA_LOCK_FAILED`。

## 9. 政策更新

本政策版本更新时,SDK 版本号会同步升级。开发者需关注 SDK 更新日志,确保其 APP 隐私政策与 SDK 实际采集行为一致。
